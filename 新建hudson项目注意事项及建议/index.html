新建hudson项目注意事项及建议

#新建hudson项目注意事项及建议：

为了安全考虑，关闭自己新建或者配置job的权限，新建或者修改设置请发送邮件到wangjunyong@doodlemobile.com或者xuming@doodlemobile.com

###注意事项：

  1. 项目名称不允许有空格
  2. 新建项目时候建议配置从其他项目copy，即Copy existing job，然后再更改git地址和更改脚本

  3. 关于脚本选择：(git上files中所能看到的文件夹为WORKSPACE/)

    1. 如果android项目直接在WORKSPACE，使用
      sample-git-14-new.sh outputApkName

    2. 如果android项目在WORKSPACE/folderName这种形式，使用
      sample-git-14-new-path.sh outputApkName folderName

    3. 如果android项目带有lib项目，比如googleservices，那么需要lib项目和android项目以（WORKSPACE/androidProjectName，WORKSPACE/libProjectName）形式放在git下面，使用（参考PoolMaster项目）
      sample-git-14-new-pathxx.sh outputApkName androidProjectName libProjectName

  4. Unity自动编译配置，基本配置如上，需要注意的是设置中需要Restrict where this job can be run选中，并指定node为slave－unity这个节点（这是另外一个机器，是mac系统） 另外脚本选择为unity-compile.sh，使用格式是：(可参考test_unity项目)
    unity-compile.sh outputApkName androidProjectFolderName libProjectFolderName

    unityFolderName
    –project
    —-androidProjectFolderName
    —-libProjectFolderName
    —-unityFolderName

  5. hudson签名功能：对于未签名包，然后上传到git上，签名脚本为：samplesignapk.sh或者samplesignmultiapk.sh
    具体使用格式是：
    samplesignapk.sh outputFileName
    需要注意是自己导出的包后缀需要为*-unsigned.apk

  6. 第一次build可能会出错，可以不予理会，再build一次

###一点建议

  1. 每次build都可以单独添加说明，这个说明与git无关
  2. 每次发布之后将发布的那个build版本点一下Keep this build forever，并且添加说明版本号码，这样就不会因为新的提交而把之前的版本删掉（hudson默认最多保存15个build，超过的时候会把之前最早的build删除，如此往复），这样可以很方便的找到以前的版本，如PoolMaster
